# 测试计划

PhotoCleaner 的手动验收测试用例。每次大功能更新后按本清单走一遍。

---

## 测试环境

| 项 | 配置 |
|---|---|
| 真机 | iPhone（任意，iOS 17.0+） |
| 模拟器 | iPhone 17 Pro，iOS 26.5（需先 `xcodebuild -downloadPlatform iOS`） |
| 测试照片库 | 至少 50 张照片，覆盖：截图、自拍、相机原图、视频、收藏、横屏、竖屏 |

---

## 1. 安装与启动

| ID | 用例 | 预期 |
|---|---|---|
| INSTALL-01 | 通过 Sideloadly 用免费 Apple ID 自签安装 IPA | App 出现在主屏 |
| INSTALL-02 | 首次启动 | 显示 `PermissionView`，文案清晰，主按钮可点 |

## 5. 滑动审核

### 5.1 进入与初始状态

| ID | 用例 | 预期 |
|---|---|---|
| SWIPE-01 | 从首页点击任一分类 | 进入 `SwipeReviewView`，加载第一张 |
| SWIPE-02 | 顶部三件套 | 左：X / 中：分类名胶囊 / 右：垃圾桶 |
| SWIPE-03 | 元数据行 | 显示「1 / 总数 · 日期 · 时间」 |
| SWIPE-04 | 卡片下方有底层卡 | 95% 缩放，60% 透明度，呈现堆叠感 |
| SWIPE-05 | 底部三按钮 | 撤销（禁用）/ 保留 / 删除（红色） |

### 5.2 左右翻页（iOS 标准方向，v0.3.0 调整）

| ID | 用例 | 预期 |
|---|---|---|
| NAV-01 | **手指向左拖** 100pt+ | **显示下一张**（currentIndex 加 1），卡片向左飞出 |
| NAV-02 | **手指向右拖** 100pt+ | **显示前一张**（currentIndex 减 1），卡片向右飞出 |
| NAV-03 | 第一张时右滑 | 卡片回弹，不切换 |
| NAV-04 | 最后一张时左滑 | 卡片回弹，不切换 |
| NAV-05 | 拖动中显示方向标签 | 左侧：「← 下一张」玻璃药丸 / 右侧：「前一张 →」玻璃药丸 / 顶部：红色「🗑 加入待删除」 |
| NAV-06 | 拖动未达阈值释放 | 卡片回弹中央 |
| NAV-07 | 拖动中卡片旋转 | 每 28pt 水平偏移 ≈ 1° 旋转 |
| NAV-08 | 触觉反馈 | 翻页时 `.light` 振动 |

### 5.3 上滑删除

| ID | 用例 | 预期 |
|---|---|---|
| DEL-01 | **手指向上拖** 130pt+ | 卡片飞出顶部，自动前进到下一张 |
| DEL-02 | 上滑中显示「↑ 加入待删除」红色标签 | 标签随 offset 渐变透明度 |
| DEL-03 | 触觉反馈 | 删除时 `.heavy` 振动（比翻页更重） |
| DEL-04 | 右上角垃圾桶 badge | 计数 +1，红色 capsule，不被裁切 |
| DEL-05 | 多次上滑 | badge 数字累加，超过 99 仍可见（不裁切） |
| DEL-06 | 撤销按钮启用 | 第一次 markDelete 后，撤销按钮从禁用变为可用 |
| DEL-07 | 上滑后顶部弹 toast | 显示「已加入待删除 · N」红色玻璃药丸，1.6 秒后自动消失 |

### 5.6 X 关闭按钮（v0.3.0 修复）

| ID | 用例 | 预期 |
|---|---|---|
| CLOSE-01 | 点击左上角 X | 立即返回首页（dismiss），触觉反馈 .light |
| CLOSE-02 | 点击 X 周边的 44pt 区域（如左上角外延 10pt） | 仍可点中（contentShape 全覆盖） |

### 5.4 撤销

| ID | 用例 | 预期 |
|---|---|---|
| UNDO-01 | 标记几张为待删除后点撤销 | currentIndex 回退，最后一张从待删除列表移除，badge -1 |
| UNDO-02 | 连续撤销直到 history 空 | 撤销按钮变灰禁用 |
| UNDO-03 | 撤销后再次上滑同一张 | 仍可重新标记 |

### 5.5 完成态

| ID | 用例 | 预期 |
|---|---|---|
| DONE-01 | 一直右滑到最后一张再右滑 | 卡片回弹（不进入完成态，因为 right=next 且 next 不可用） |
| DONE-02 | 一直上滑直到所有都被标记 | 显示完成态：「已审核完成」绿色徽章 + 「查看待删除列表」按钮 |
| DONE-03 | 完成态点击「查看待删除列表」 | 打开 PendingDeletionView |

## 6. 待删除列表

| ID | 用例 | 预期 |
|---|---|---|
| PEND-01 | 点击右上角垃圾桶 | 弹出 sheet `PendingDeletionView` |
| PEND-02 | 网格显示所有待删除缩略图 | 自适应列数，每张右上角有 × 按钮 |
| PEND-03 | 点击单张缩略图 ×  | 该张从列表移除，badge 同步 -1 |

| PEND-09 | 列表清空时进入 sheet | 显示「暂无待删除照片」空状态 |

## 9. 边界与异常

| ID | 用例 | 预期 |
|---|---|---|
| EDGE-01 | 照片库为空 | 进入分类显示「加载中…」永远转圈是 bug，应改为空状态提示 |
| EDGE-02 | 某分类无匹配项 | 「加载中…」转 1 秒后切换为「已审核完成」无标记态 |
| EDGE-03 | 网络相册（iCloud 未下载） | 触发自动下载，期间显示 ProgressView |
| EDGE-04 | 删除过程中关闭 App | 重启后状态不丢失（pendingDeletion 暂存在 VM，重启会清空——已知限制） |
| EDGE-05 | 同一张照片被删除两次 | 第二次系统拒绝（PhotosKit 抛错），App 应 graceful 处理 |

## 10. 回归测试矩阵

每次发版前至少跑这 10 个用例：

1. INSTALL-02
2. PERM-01
3. HOME-04（segmented control 切换）
4. NAV-01（左滑前一张）
5. NAV-02（右滑下一张）
6. DEL-01（上滑加入待删除）
7. DEL-04（badge 不被裁切）
8. PEND-06（系统原生删除对话框）
9. UNDO-01（撤销）
10. PERF-02（连续滑动 60fps）

通过率应为 100%，任一失败阻断发版。
